# Телеграм‑бот сервиса ремонта (Оренбург)

**Назначение**: бот для записи / запросов по ремонту мобильных телефонов и другой техники, с инлайн‑кнопками и простым админ‑меню для редактирования текстов прямо в боте.

## Функции бота

- **Старт**: при `/start` бот показывает приветствие и главное меню.
- **Выбор услуги** (все через inline‑кнопки):
  - **Ремонт / диагностика** — полноценная запись:
    - выбор категории (кнопки по 3 в ряд): `Apple`, `Samsung`, `Huawei / Honor`, `Xiaomi`, `Meizu`, `Nokia`, `Asus`, `ZTE`, `Lenovo`, `LG`, `Наушники`, `Ноутбуки`;
    - выбор модели (также по 3 в ряд, для ноутбуков список из ТЗ);
    - выбор проблемы: `Экран`, `Аккумулятор`, `Не включается`, `Чистка`, `Увеличение`, `Клавиатура`, `Прошивка`, `После влаги`, `Другое`;
    - по выбранной проблеме бот показывает описание/стоимость, затем собирает доп.описание;
    - дальше: выбор офиса → день (`Завтра`/`Другой день`) → дата (если нужно) → слот (полчаса) или `Приду в течение дня` → имя → телефон/`@username`;
    - в конце бот отправляет подтверждение в стиле: «записал на дату/время...» и возвращает в главное меню;
    - админу уходит полная заявка.
  - **Гидрогелевая плёнка** — показывает текст из `config.json`.
  - **Стоимость защиты и аксессуаров** — показывает текст из `config.json` (Оренбург).
  - **Моё устройство в сервисе**:
    - сначала просит выбрать офис;
    - затем просит ввести номер заказа `XXX-XXXXX` или телефон `79999999999`;
    - если это телефон — спрашивает, доступен ли по нему (Да/Нет);
    - если **Да** — всё отправляется админу;
    - если **Нет** или введено не по формату — просит контакт (телефон или @username) и отправляет всё админу.
  - **Хочу обратиться по гарантии** — показывает гарантийный текст из `config.json`.
- **Админу всё прилетает в личку** (ID администратора задаётся в `.env`).

## Админ‑панель

- Доступна по команде **`/admin`** только пользователям из `ADMIN_IDS`.
- Имеет 3 раздела: **Тексты**, **Json**, **Шаблоны**.
- Можно менять:
  - обычные тексты (приветствие, гарантия, гидрогель, подсказки);
  - списки/структуры в формате JSON:
    - `repair_categories`,
    - `repair_models_by_category`,
    - `repair_problems_by_category` (разные проблемы для каждой категории устройства),
    - `repair_laptop_models`,
    - `repair_problem_texts` (описания и цены услуг),
    - `offices`, `office_addresses`,
    - `repair_time_slots`.
- Для JSON-полей отправляйте валидный JSON одним сообщением.

## Установка и запуск

1. **Создайте и активируйте виртуальное окружение (по желанию)**:

```bash
cd tel_remont
python -m venv .venv
.venv\Scripts\activate
```

1. **Установите зависимости**:

```bash
pip install -r requirements.txt
```

1. **Создайте файл `.env`** на основе `.env.example`:

```text
BOT_TOKEN=ваш_токен_бота_от_BotFather
ADMIN_IDS=ваш_telegram_id_1,ваш_telegram_id_2
```

1. **Отредактируйте `config.json` при необходимости** (адреса офисов, тексты и т.п.) или меняйте тексты через `/admin` в самом боте.

1. **Запустите бота**:

```bash
python bot.py
```

Убедитесь, что у бота включён режим **Receive messages** и он не запущен где‑то ещё (одновременно бот может работать только на одном сервере).

## Docker Compose

1. Убедитесь, что заполнен `.env` (как в примере выше).
1. Запустите контейнер:

```bash
docker compose up -d --build
```

1. Посмотреть логи:

```bash
docker compose logs -f bot
```

1. Остановить:

```bash
docker compose down
```
